# 🧭 Карьерный компас (Career Compass)

«Карьерный компас» — это интеллектуальный веб-сервис для профориентации в сфере IT, автоматического подбора подходящих профессий и автоматизированного построения персонализированных интерактивных дорожных карт (Roadmaps) развития специалиста с интеграцией LLM-наставника.

Сервис использует комбинированный алгоритм машинного обучения: анализирует развернутые текстовые ответы пользователя на открытые вопросы с помощью NLP-эмбеддингов и сопоставляет его профиль личностных качеств (traits) через многомерный векторный поиск.

---

## ✨ Основные возможности

* **Интеллектуальное тестирование**: Сбор информации через закрытые вопросы (психометрические свойства и гибкие навыки) и открытые развернутые вопросы о целях, ценностях и идеальном рабочем дне.
* **Гибридный рекомендательный пайплайн (ML)**: 
    * Семантический анализ открытых ответов с использованием компактной языковой модели `rubert-tiny-turbo` (архитектура Transformer).
    * Многомерный векторный поиск ближайших соседей (KNN) по эталонным профилям профессий.
    * Вычисление финальной уверенности (Confidence Score) на основе взвешенного слияния текстовых и трейтовых метрик сходства.
* **Интерактивные Roadmaps**: наличие планов обучения для каждой рекомендованной IT-специальности (в будущем).
* **LLM-Ассистент**: интеграция со Сбер GigaChat API для автоматического детального объяснения сложных тем из дорожной карты в один клик.
* **Панель Администратора**: интерфейс для управления опросниками и настройки контента дорожных карт.
* **Безопасность**: Асинхронная аутентификация на основе JWT-токенов (с парой Access/Refresh) и защищенное хранение паролей.

## 🛠 Технологический стек

### Бэкенд (Backend)
* **Язык**: Python 3.13
* **Фреймворк**: **FastAPI** (полностью асинхронный инференс и обработка запросов)
* **База данных**: SQLite (асинхронный движок через **SQLAlchemy** + **aiosqlite**)
* **Машинное обучение (ML)**: `sentence-transformers` (`rubert-tiny-turbo`), `scikit-learn` (алгоритм `NearestNeighbors`)
* **Интеграция с ИИ**: `httpx` для асинхронного взаимодействия с **GigaChat API** (с кастомной конфигурацией защищенного SSL-соединения через корпоративный Bundle-сертификат)

### Фронтенд (Frontend)
* **Сборщик**: **Bun** (современный менеджер пакетов и сборщик)
* **Библиотека**: **React**
* **Стейт-менеджмент и роутинг**: Экосистема **TanStack** (TanStack Router)
* **Стилизация**: **TailwindCSS** (адаптивная и лаконичная верстка интерфейса)

### Деплой и DevOps
* **Контейнеризация**: Docker / Docker Compose
* **Веб-сервер**: Nginx (для дистрибуции статики фронтенда в production-окружении)

## 🚀 Быстрый запуск (Развертывание)

Самый быстрый способ запустить проект целиком (Бэкенд + Фронтенд + БД) в идентичном продакшену окружении — использовать Docker Compose.

### 1. Подготовка конфигурации окружения

Создайте файл `.env` в корневом каталоге проекта и укажите ваши настройки (особенно секретный ключ GigaChat):

```env
LLM_API_KEY=Ваш_Basic_Auth_Токен_GigaChat
JWT_SECRET=super-duper-secret-key-change-me-in-production
ACCESS_TOKEN_EXPIRE_MINUTES=60
REFRESH_TOKEN_EXPIRE_DAYS=7
ALGORITHM=HS256
DATABASE_URL=sqlite+aiosqlite:////app/data/database.db

```

### 2. Запуск контейнеров

Запустите сборку и старт всех сервисов одной командой:

```bash
docker compose up --build

```

После успешного выполнения команды:

* **Фронтенд** (Разработка с hot-reload) будет доступен по адресу: `http://localhost:5173`
* **Бэкенд API** будет доступен на: `http://localhost:8000`
* **Интерактивная документация API (Swagger UI)**: `http://localhost:8000/docs`

## 📈 Спецификация ML-модели

Модуль `CareerRecommender` использует два параллельных механизма оценки:

1. **Text Score**: Косинусное сходство между эмбеддингом объединенных ответов пользователя и эталонными текстовыми векторами, сформированными моделью `deepvk/rubert-tiny-turbo`.
2. **Trait Score**: Расстояние Минковского/Евклида между нормализованными векторами личностных шкал пользователя и матрицы требований профессий.

Итоговое ранжирование выполняется по формуле:


$$\text{Confidence} = w_{\text{text}} \cdot \text{Sim}_{\text{text}} + (1 - w_{\text{text}}) \cdot \text{Sim}_{\text{trait}}$$


По умолчанию вес текста $w_{\text{text}} = 0.6$. Настройка отсечения шума (пустых или неадекватных ответов пользователя) регулируется динамическим порогом схожести.
